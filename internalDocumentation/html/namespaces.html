SoXPlugins
Reimplementation of SoX as DAW Plugins
Namespace List
Here is a list of all documented namespaces with brief descriptions:
[detail level 123]
 ▼NSoXPluginsThe package SoXPlugins provides all types for the DAW plugins emulating the command-line audio processor SoX; the connection to the plugin host is abstracted via the JUCE framework, the effect implementation itself is completedly self-contained
 ▼NBaseTypesThe package BaseTypes provides the elementary types used for the SoX plugins; those are primitive types (like strings or reals), containers (like generic tuples or matrices) and modules for assertion and logging management
 NContainersThe package Containers provides the containers used for the SoX plugins (like lists of string or reals or a dictionary etc.)
 NPrimitivesThe package Primitives provides the primitive types used for the SoX plugins; those are type wrappers around booleans, integers, naturals, percentages, objects, radians, reals and strings
 NStringUtilThe StringUtil package provides several string routines not included in the standard C++ library (like, for example, prefix calculation, whitespace stripping etc)
 ▼NCommonAudioThe package CommonAudio contains the audio specific types used for the SoX plugins (like sample management, IIR filtering, waveform management etc.)
 NSoXAudioHelperThe package SoXAudioHelper defines several audio helper routines (like, for example, converting decibels to linear factor or a range adjustment for a percentage value)
 ▼NEffectsThe package Effects provides types for the different audio effects themselves; the currently supported SoX effects are compander, filter, gain, overdrive, phaser/tremolo and reverb
 NSoXCompanderThe package SoXCompander provides classes for the SoX compand and mcompand audio effects
 NSoXFilterThe package SoXFilter provides classes for the SoX audio effects for IIR filters; those are allpass, band, bandpass, bandreject, bass, biquad, equalizer, highpass, lowpass and treble filters that are combined into a single plugin
 NSoXGainThe package SoXGain provides classes for the SoX gain audio effect
 NSoXOverdriveThe package SoXOverdrive provides classes for the SoX overdrive audio effect
 NSoXPhaserAndTremoloThe package SoXPhaserAndTremolo provides classes for the SoX audio modulation effects phaser and tremolo that have similar parameters and hence are combined into a single plugin
 NSoXReverbThe package SoXReverb provides classes for the SoX reverb audio effect
 NViewAndControllerThe package ViewAndController provides classes for the interconnection to the enclosing plugin framework (abstracted via the JUCE framework): a SoXAudioProcessor connects to the plugin host, the SoXAudioEditor is the UI component and interacts with the user and consists of SoXAudioEditorWidgets like sliders and comboboxes to show and get data
